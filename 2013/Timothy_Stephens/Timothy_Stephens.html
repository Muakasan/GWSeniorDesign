GW Senior CS Design 2013

Timothy Stephens: MARRC

MARRC-MULTIPLE AUTONOMOUS REMOTE ROBOT COORDINATION SYSTEM: A design project byThe MARRC system is a robotic task/responsibility assignment system to allow a group of robots to work as a cohesive unit towards a common goal.

The MARRC system began as an idea to cooperatively map an area. As the project progressed, it shifted to a different angle. The current design scheme of the MARRC System will be to fulfill a security protocol, with emphasis on error recovery. This means that all MARRC-bots in the system will cooperatively cover an area, emulating the actions of a personal security team. When one "breaks" (sends an error code, or shuts down), the others will respond and cover the region previously covered by the broken MARRC-bot. At a signal, as if a gun were fired upon the security team's protected asset, the MARRC-bots will converge on that asset and cover it to a designated extraction point.

UPDATE:

MARRC System is implemented as a mapping protocol, with potential security applications. Path tracking and obstacle detection are in place so that other protocols could be added if desired.Below are two pictures of a single MARRC-bot, the platform the MARRC system will run on. Each MARRC-bot consists of an iRobot Create, a netbook, and a webcam. The system will be able to run in either configuration, netbook open or closed.
